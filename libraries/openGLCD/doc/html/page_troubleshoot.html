Troubleshooting
  • No pixels visible on the display
    1. Check all solder connections to GLCD module (all connections to PCB must be soldered)
    2. Check +5v and Gnd connections between Arduino and GLCD panel
    3. Check all data and command pins are wired according to the datasheet and matching the configuration settings (This is the most common cure to this problem)
    4. Check the datasheet for your panel to verify appropriate timing values are set in the configuration file
    5. Check contrast voltage (typically between -3 and -4 volts) on contrast-in pin of LCD panel While the sketch is operating, try gradually adjusting the pot through its range Some displays are very sensitive to this setting
    6. Check sketch has compiled ok and downloaded to Arduino
    7. Run GLCDDiags test sketch.
  • All pixels are dark
    1. Check contrast voltage (typically between -3 and -4 volts) on contrast-in pin of LCD panel. While the sketch is operating, try gradually adjusting the pot through its range. Some displays are very sensitive to this setting
  • Left and right side of image reversed
    1. Swap CSEL1 and CSEL2 wires (or swap pin assignments in header file)
  • No Serial port diagnostic information
    1. Verify the code is compiling and properly uploading
    2. Disonnect glcd wires to ensure that no pins are in conflict with the serial port pins
